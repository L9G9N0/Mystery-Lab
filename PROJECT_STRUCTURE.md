## 🎯 Directory Descriptions & Responsibilities

### 🟢 `api/`
This folder is used exclusively by **Vercel** to host the Python serverless functions.
* `index.py`: Serves as the ASGI handler which intercepts requests to `/api/*`, strips the `/api` prefix, and routes the request into our FastAPI application.

### 🟢 `app/`
Contains the Python-based FastAPI backend engine.
* `api/`: Holds routing endpoints for core modules (bookings, newsletter, feedback, admin dashboard).
* `core/`: Loads configurations dynamically from `.env` using Pydantic Settings, handles validation on startup, and exports the `supabase` database proxy.
* `schemas/`: Implements Pydantic schemas which validate incoming JSON payloads (e.g. validating email strings, phone number patterns).
* `main.py`: Declares the FastAPI app instance, adds CORS middleware supporting localhost and production vercel subdomains, and mounts the routers.

### 🟢 `migrations/`
Holds data definition script files.
* `supabase_schema.sql`: Contains the DDL instructions to create the SQL schema for Supabase, configures Row Level Security (RLS) policies on all tables, and inserts initial seed data.

### 🟢 `src/`
Hosts the frontend single-page application built on Vite and React.
* `components/`: UI components organized as self-contained files. Layout modules like `Header` handle navigation and dynamic user profile display, and core sections like `ScienceKits` display options with modular forms.
* `lib/`: Houses framework clients (like Supabase client instantiator) and utility style injectors (`cn`).
* `store/`: Handles state providers (like `AuthContext.tsx` which listens to auth sessions and manages user profiles).

### 🟢 `tests/`
Houses unittest files which verify API correctness.
* Includes mock-assert unit tests (`test_api.py`) and integration tests (`test_verify_mystery_lab.py`) that verify database queries, token validation, and administrative route access blocks.
